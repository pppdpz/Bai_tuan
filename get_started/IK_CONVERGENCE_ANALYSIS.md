# IK 收敛性分析与解决方案

## 问题诊断

### FK 分析结果（真实结构）

通过正运动学计算，左侧太阳能板的真实坐标为：

```
卫星主箱体中心：     Y = 0.60
折叠臂连接根部：     Y = 0.52, Z = 1.1
垂直立柱折叠区：     Y = 0.42 (Panel 1-3 折叠在同一垂直面)
最外侧悬空端：       Y = 0.24, Z = 1.28 (Panel 4 水平伸出的"舌头")
```

### 关键发现

1. **Y=0.24 是正确的最外端目标**（panel_4_L 的悬空尖端）
2. **Y=0.55 是错误的**（靠近主体根部，会抓在内侧）
3. **之前的"悖论"**：设置 Y=0.24 却抓在内侧，是因为 IK 求解器失败，停在了 Y≈0.46

## IK 求解器失败的原因

### 可能原因 1：容差设置过宽
- 求解器接受了"接近"目标的解（Y=0.46），而不是真正到达目标（Y=0.24）
- 差距：22cm（0.46 - 0.24 = 0.22m）

### 可能原因 2：关节限位
- UR5e 的关节限位可能阻止机械臂完全伸展到 Y=0.24
- 需要检查 shoulder_lift_joint 和 elbow_joint 的限位

### 可能原因 3：碰撞检测
- 求解器可能检测到与卫星主体的潜在碰撞
- 在 Y=0.46 处停止以避免碰撞

### 可能原因 4：初始猜测不佳
- 从当前位置到目标位置跨度太大
- 求解器陷入局部最优解

## 解决方案

### 方案 1：验证 IK 收敛性（已实现）

在 `4_motion_planning_baituan.py` 中添加了 IK 收敛性验证：

```python
# 在关键步骤验证 IK 是否真正到达目标位置
if step in [0, 50, 70, 90, 120, 200, 220, 250]:
    # 临时应用求解结果验证实际到达位置
    _pinch_pos_reached = _data.site_xpos[pinch_site_id].copy()
    _target_pos = ee_pos_batch_world[0].cpu().numpy()
    
    # 计算位置误差
    pos_error = np.linalg.norm(_pinch_pos_reached - _target_pos)
    y_error = abs(_pinch_pos_reached[1] - _target_pos[1])
    
    # 报告误差
    if y_error > 0.05:  # 50mm 阈值
        log.error(f"❌ Y 轴误差过大！未到达最外端悬空点")
```

### 方案 2：调整 IK 求解器参数

需要找到 IK 求解器的配置并调整：

```python
# 示例（具体参数取决于使用的求解器）
ik_solver_config = {
    'max_iterations': 200,      # 增加迭代次数
    'tolerance': 1e-4,          # 降低容差（更严格）
    'step_size': 0.1,           # 调整步长
    'use_nullspace': True,      # 使用零空间优化
}
```

### 方案 3：分阶段逼近

将大跨度运动分解为多个小步骤：

```python
# 在 get_trajectory_targets 中添加中间点
waypoints = [
    [0.3, 0.18, 1.20],  # 初始位置附近
    [0.3, 0.20, 1.25],  # 中间点 1
    [0.3, 0.22, 1.27],  # 中间点 2
    [0.3, 0.24, 1.28],  # 最终目标
]
```

### 方案 4：检查关节限位

检查 MJCF 文件中的关节限位设置：

```xml
<joint name="shoulder_lift_joint" range="-3.14159 0" />
<joint name="elbow_joint" range="-3.14159 3.14159" />
```

如果限位过严，可能需要调整。

### 方案 5：使用更好的初始猜测

```python
# 使用上一步的解作为初始猜测（已实现）
curr_q = obs.robots[robot.name].joint_pos[:, inv_idx]

# 或者使用预计算的"接近解"
if step == 50:  # 接近阶段开始
    curr_q = torch.tensor([0.3, -0.8, 1.5, -2.2, -1.57, 0.0])
```
